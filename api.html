API Documentation

Saints.Today will offer an API to enable you to include saint content directly on your website. API documentation is forthcoming.

GET the Saint of the Day

To retreive the data for the Saint of the Day, a GET request can be made like this:

https://saint.wpengine.com/wp-json/wp/v2/saint?filter[monthnum]=1&filter[day]=1

The first "1" in the string (following [monthnum]) should be the number of the month (1-12). The second "1" in the string (following [day]) should be the number of the day. Setting each of these (the month and day) as pre-populated variables, dependent on the current date, will enable you to load the saint of the day each day.

Here is an example ajax request, which relies upon JavaScript to first ascertain today's date:

var today = new Date();
var day = today.getDate();
var month = today.getMonth();
month = month + 1; // Because JavaScript months start at 0

$.ajax({
  type: "GET",
  url: 'https://saint.wpengine.com/wp-json/wp/v2/saint?filter[monthnum]=' + month + '&filter[day]=' + day,

Using PHP, if you set the root element as "post" you'd be able to access the key data points this way:

post.acf.title // Saint name
post.title.rendered // Saint title (Saint or Saints or Pope Saint)
post.date // The unformatted date
post.acf.text // The saint biography
post.acf.reflection // The saint reflection
